# 需求说明

## 项目目标

构建一个网页版热点监测系统，先完成可运行网页产品并通过验收，再开发 Agent Skills。

系统需要定时从多个来源获取最新信号，利用 AI 识别热点，并在热点达到阈值时推送提醒。

## 信息源

固定采用 8 个信息源，避免单一来源导致误判：

1. Twitter / X
2. Bing
3. Google
4. DuckDuckGo
5. HackerNews
6. 搜狗
7. B 站
8. 微博

## 数据获取要求

- Twitter / X：对接 `https://twitterapi.io/` 获取最新信息。
- 其他来源：后端通过网页搜索 URL 进行低频抓取，不要求使用官方 API。
- 需要注意访问频率，默认每 30 分钟检查一次热点。
- 应支持手动立即分析，用于调试和人工验收。

## AI 要求

- 使用 OpenRouter 接入 AI 服务。
- AI 负责综合多源信息并识别热点。
- 输出应包含热点等级、热度分、关键词、证据来源、风险、推荐动作等结构化字段。
- 没有 OpenRouter Key 时，系统可以提供启发式回退评分，便于本地验证基础流程。

## 推送要求

- 浏览器推送：页面打开并授权后，当热点达到阈值时弹出浏览器通知。
- 邮箱推送：可选，用户配置 SMTP 后启用。
- 推送阈值默认按热度分判断。

## 前端要求

- 网页需要足够独特，不做千篇一律模板页。
- 页面需要支持配置关键词、查看 8 个来源状态、人工补充来源文本、查看 AI 分析结果。

## 开发顺序

1. 沉淀需求文档与技术方案。
2. 利用 MCP 获取最新 API 对接和技术实现方法。
3. 开发网页版并完成测试。
4. 找用户验收网页版。
5. 验收通过后开发 Agent Skills。
